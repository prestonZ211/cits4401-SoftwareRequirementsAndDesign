## User Stories

The following three user stories were developed based on the stakeholders identified in Task A. Each story covers a different stakeholder perspective and follows the standard format:  
*As a [role], I want [functionality], so that [business value].*

---

### US-B01 — Researcher: Submit an Application

> As a researcher, I want to submit my ethics application and all supporting documents through a single online system, so that I no longer have to manage email chains and scattered file attachments just to start the review process.

**Context:** Currently, researchers complete application forms as Word documents and email them along with attachments such as consent forms and risk assessments. This makes it difficult to track what has been submitted and leads to version confusion. A unified submission system would give researchers a clear, organised way to lodge everything in one place.

---

### US-B02 — Ethics Committee Member: Access Application Materials

> As an ethics committee member, I want all submitted application materials to be available in one centralised location, so that I can review everything I need without having to request documents from multiple sources.

**Context:** At present, committee members receive documents via email or have to retrieve them from shared folders, which are not always up to date. This makes the review process slow and inconsistent. Having all materials accessible in one system would allow reviewers to start their work immediately and be confident they are looking at the correct version of each document.

---

### US-B03 — Administrative Staff: Monitor Application Progress

> As an administrative staff member, I want a dashboard showing the current status of every active application, so that I can quickly identify which applications are delayed or require follow-up without manually checking spreadsheets.

**Context:** Administrative staff currently track application progress using spreadsheets updated manually, which is time-consuming and prone to error. A real-time status dashboard would allow staff to see at a glance where each application is in the process, reducing the risk of applications being overlooked or delayed without anyone noticing.
