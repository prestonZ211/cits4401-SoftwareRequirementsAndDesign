## User Stories

The following three user stories were developed based on the stakeholders identified in Task A. Each story covers a different stakeholder perspective and follows the standard format:  
*As a [role], I want [functionality], so that [business value].*

---

### US-B01 — Researcher: Submit an Application

> As a researcher, I want to submit my ethics application and all supporting documents through a single online system, so that I no longer have to manage email chains and scattered file attachments just to start the review process.

---

### US-B02 — Ethics Committee Member: Access Application Materials

> As an ethics committee member, I want all submitted application materials to be available in one centralised location, so that I can review everything I need without having to request documents from multiple sources.

---

### US-B03 — Administrative Staff: Monitor Application Progress

> As an administrative staff member, I want a dashboard showing the current status of every active application, so that I can quickly identify which applications are delayed or require follow-up without manually checking spreadsheets.

---

## Notes

- US-B01 covers the **Researcher** stakeholder
- US-B02 covers the **Ethics Committee Member** stakeholder
- US-B03 covers the **Administrative Staff** stakeholder
- These stories are intentionally distinct to complement those written in Tasks C and D
